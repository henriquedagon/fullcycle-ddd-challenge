Entities:
 - Uma entidade é algo único que é capaz de ser alterado de forma contínua por um longo periodo de tempo.
 - Uma entidade é algo que possui uma continuidade em seu ciclo de vida e pode ser distiguida 
    independente dos atributos que são importantes para a aplicação do usuário.
    Pode ser uma pessoa, um carro, cidade, um ticket de loteria ou uma transação bancária.
 - Entidade = IDENTIDADE
 - (Uma entidade tem que estar sempre válida)

Value objects
 - Quando você se preocupa apenas com atributos de um elemento, de um model, classifique isso como um 
    value object.
 - Trate o value object como imutável.
 - (Dar expressividade)

Aggregate
 - Um agregado é um conjunto de objetos associados que tratamos como uma unidade para um propósito
    de mudança de dados.

Domain services
 - Um serviço de domínio é uma operação sem estado que cumpre uma tarefa específica do domínio.
    Muitas vezes, a melhor indicação de que você deve criar um Serviço no modelo de domínio é que quando
    a operação que você precisa executar parece nao se encaixar como um método em um Agregado ou um 
    Objeto de Valor
 - Quando um processo ou transoformação significativa no domínio nao for uma responsabilidade natural
    de uma Entidade ou Objeto de Valor, adicione uma operação ao modlelo como uma interface autônoma 
    declarada como um Serviço. Defina a interface em baseada na linguagem do modelo de domínio e 
    certifique-se de que o nome da operação faça parte do Ubiquituous Language. Torne o Serviço sem estado.

Repository
 - Um repositório comumente se refere a um local de armazenamento, geralmente considerado um local de 
    segurança ou preservação dos itens nele armazenados. Quando você armazena algo em um repositório e 
    depois retorna para recuperá-lo, você espera que ele esteja no mesmo estado que estava quando você o
    o colocou lá. Em algum momento, você pode optar por remover o item armazenado do repositório.
 - Esses objetos são semelhantes a coleções sobre persistência. Todo agregado presistente terá um repositório.
    De um modo geral, há uma relação de um-para-um entre um tipo de agregado e um repositório.
